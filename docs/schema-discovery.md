# PowerShop 4D Database Schema Discovery

> Full schema reference for the PowerShop POS / ERP database running on 4D Server.
> Generated by querying `_USER_TABLES`, `_USER_COLUMNS`, `_USER_VIEWS`, and `_USER_IND_COLUMNS` system tables,
> and by string-extraction analysis of the compiled application structure (`PowerShop.4DC`).
> Last updated: 2026-04-05.

## Overview

| Metric | Value |
|--------|-------|
| Total tables | 324 (confirmed from compiled structure labels.json) |
| Tables with data | ~102 |
| Total queryable columns | 16,179+ |
| SQL Views defined | 100 (50 `*_SQL` + 50 `*_BI`) — see [sql-views.md](sql-views.md) |
| Largest table (rows) | Exportaciones (~2,058,201) |
| Widest table (columns) | Preferencias3 (1,124 cols) |

## Schema Sources

| Source | Method | Value |
|--------|--------|-------|
| `_USER_TABLES` / `_USER_COLUMNS` | SQL query via p4d | Table/column names, types, nullability |
| `_USER_VIEWS` | SQL query via p4d | 100 SQL views (50+50 BI) — vendor's intended query patterns |
| `_USER_IND_COLUMNS` | SQL query via p4d | Indexed columns per table — confirms access patterns |
| `PowerShop.4DC` strings | `strings -n 5` on 360 MB compiled binary | 5.7M string lines — table/field/method names, form structures |
| SQL views (`*_SQL`) | Direct `SELECT * FROM <view> LIMIT 1` | Exact column lists for all analytics-relevant views |
| SOAP WSDL | zeep introspection | 130 WS_JS_* + 934 WS_* method signatures |

### Key Queries Used

```python
# venv: /Users/alobato/git/powershop-analytics/.venv/bin/python3
import p4d, json

conn = p4d.connect(host='10.0.1.35', port=19812, user='Administrador', password='')
cur = conn.cursor()

# 1. All tables and column metadata
cur.execute("SELECT TABLE_NAME, COLUMN_NAME, TYPE_NAME, IS_NULLABLE FROM _USER_COLUMNS ORDER BY TABLE_NAME, ORDINAL_POSITION")
rows = cur.fetchall()
by_table = {}
for r in rows:
    by_table.setdefault(r[0], []).append(r[1])
json.dump(by_table, open('/tmp/4d_all_columns.json', 'w'))

# 2. All SQL views
cur.execute("SELECT TABLE_NAME FROM _USER_VIEWS ORDER BY TABLE_NAME")
views = [r[0] for r in cur.fetchall()]
# For each view: SELECT * FROM <view> LIMIT 1 to get column descriptions
# (run one view per process — wide views with Picture/Blob crash the driver)

# 3. All indexed columns
cur.execute("SELECT TABLE_NAME, COLUMN_NAME, INDEX_NAME FROM _USER_IND_COLUMNS ORDER BY TABLE_NAME")
rows = cur.fetchall()
json.dump(rows, open('/tmp/4d_index_columns.json', 'w'))

# 4. FK/PK constraints
cur.execute("SELECT TABLE_NAME, COLUMN_NAME, CONSTRAINT_NAME, CONSTRAINT_TYPE FROM _USER_CONS_COLUMNS ORDER BY TABLE_NAME")
rows = cur.fetchall()
json.dump(rows, open('/tmp/4d_cons_columns.json', 'w'))
```
